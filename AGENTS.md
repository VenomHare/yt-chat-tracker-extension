# Agent Instructions

## Architecture Changes

Any change to the system architecture, design patterns, or component relationships must be documented in `SPEC.md`. This includes:

- New files or directories added to the project structure
- Changes to existing file purposes or responsibilities
- New dependencies or integrations
- API design changes
- Database schema changes

## Known Bugs

Any discovered bugs, limitations, or known issues must be documented in `BUGS.md`. Include:

- Bug description
- Workaround if available
- Severity/impact

## Workflow

1. Before making architectural changes: Update `SPEC.md` with the planned changes
2. When discovering bugs: Update `BUGS.md` immediately when identified
3. After completing changes: Verify both files reflect current state
